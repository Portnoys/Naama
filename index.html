כל אחד יכול INSERT ו-SELECT מוגבל. כללי דוגמה: // 1) מדיניות INSERT לכל: using (true) with check (true) // 2) מדיניות SELECT לכל: using (true) // אם נדרש פרטיות גבוהה יותר, אפשר לבטל SELECT לציבור ולקרוא נתונים דרך Key בצד אדמין בלבד. // ============================= // ▶️ הוראות פריסה ל-GitHub Pages // ============================= // 1) יצירת ריפו בגיטהאב והעלאת שני קבצים לשורש: index.html, admin.html // 2) Settings → Pages → Source: Deploy from a branch → בחרי את הסניף (למשל main) ואת התיקיה /root → Save // 3) המתיני לדף ה-Pages להיבנות. ה-URL יהיה בד"כ: https://.github.io// // 4) פתחי את index.html עם פרמטרים: ?v=https://.../video.mp4&pid=S01 // 5) הדביקי בקובץ index.html את SUPABASE_URL ו-ANON_KEY שלך. // 6) לצפייה בתוצאות, פתחי admin.html והזיני את אותם פרטים (או תקבעי אותם ישירות בקוד). // ✅ טיפים // - אפשר לשים את ה-ANON KEY בקובץ נפרד ולכלול אותו בעמוד, אבל ב-GitHub Pages אין סודיות אמיתית בצד לקוח. זה בסדר למחקר ציבורי ללא מידע מזהה. // - אם צריך פרטיות חזקה: השאירי INSERT פתוח ו-SELECT חסום לציבור, וב-Admin תשתמשי בקובץ מקומי עם key בעל הרשאות מתאימות, או פרסיון פרטי. // - לחלופין אפשר להשתמש ב-Google Apps Script שמריץ כתיבה ל-Google Sheets ומסתיר את ה-Key; אם תרצי, אכין לך גם גירסה כזו. // - אפשר להנגיש לכל נבדק לינק עם אותו וידאו ו- pid שונה, או להחליף את ה-URL של הסרטון לכל משימה. // - אם תרצי לחסום דילוג קדימה, אפשר להגדיר controlsList="nodownload noplaybackrate noremoteplayback" ולנטר seeked. // - אפשר להוסיף שדות נוספים (למשל הערה טקסטואלית) — בצד הלקוח להוסיף input ובשרת להרחיב את הסכמה. // - לשמירה מאובטחת בענן, אפשר לחבר ל-Supabase/Firebase/Google Sheets—אשמח להוסיף קוד בהתאם לצורך.
